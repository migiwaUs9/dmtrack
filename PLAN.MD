\# デュエプレ自動戦績・棋譜記録システム 開発仕様書



\## 1. プロジェクト概要

PC版「デュエル・マスターズ プレイス」のプレイ画面を常時モニタリングし、完全非同期・非干渉（Read-Only）でゲームの進行状態を推論する個人用ツール。使用されたカードの時系列データ（棋譜）の作成、対面デッキの自動推定、戦績データの外部ファイル出力を行う。

\*\*※重要制約:\*\* 本ツールは個人的な技術検証および個人でのゲームプレイの利便性向上のためのものであり、商用目的ではない。



\## 2. 絶対遵守ルール（セーフティガイドライン）

\- \*\*完全なRead-Only\*\*: ゲームプロセスのメモリへの干渉、通信パケットの傍受、ゲームクライアントの解析・改ざん、および自動クリック等の入力（Bot/マクロ操作）は\*\*いかなる場合も実装しない\*\*。

\- 画面のピクセルデータ（スクリーンショット）の取得と、その画像解析のみでシステムを完結させること。



\## 3. 技術スタック

\- \*\*言語:\*\* Python 3.x

\- \*\*画面キャプチャ:\*\* `mss` （低負荷・高速）

\- \*\*画像処理・状態判定:\*\* `opencv-python` (`cv2`)

\- \*\*文字認識 (OCR):\*\* `pytesseract` または `EasyOCR`

\- \*\*データ処理:\*\* `pandas` または標準 `csv` モジュール



\## 4. モジュール構成と状態遷移（ステートマシン）

メインループ内で以下の状態（State）を管理し、画像トリガーによって遷移させる。



\### State A: 待機状態 (Out of Match)

\- \*\*処理:\*\* 画面全体を低頻度（例: 3秒に1回）でキャプチャ。

\- \*\*トリガー:\*\* 「VS画面」またはバトル画面特有の固定UIを検知。

\- \*\*遷移:\*\* State Bへ移行。内部の「棋譜ログ」と「デッキ推定スコア」を初期化。



\### State B: ターン監視状態 (Turn Tracking)

\- \*\*処理:\*\* 画面中央の帯UI領域を高頻度（例: 1秒に2回）で監視（テンプレートマッチング）。

\- \*\*トリガー1:\*\* 「YOUR TURN (青)」検知 → 内部状態を `Current\_Player = Me` に更新し、ターン数をインクリメント。

\- \*\*トリガー2:\*\* 「ENEMY TURN (赤)」検知 → 内部状態を `Current\_Player = Enemy` に更新。

\- \*\*トリガー3:\*\* 画面中央に「カード拡大表示」のエフェクトを検知。

\- \*\*遷移:\*\* トリガー3発火で State C へ移行。



\### State C: カード検知・OCR状態 (Card Play Detection)

\- \*\*処理:\*\* 拡大表示されたカード領域をクロップし、二値化等の前処理を経てOCRエンジンでテキスト（カード名）を取得。

\- \*\*アクション:\*\* - `\[Turn\_Count, Current\_Player, Card\_Name]` の形式で棋譜ログに追記。

&#x20; - `Current\_Player = Enemy` の場合、取得したカード名をデッキ推定エンジンに渡しスコアを更新。

\- \*\*遷移:\*\* 拡大表示が消えたら State B へ戻る。



\### State D: リザルト検知 (Result)

\- \*\*トリガー:\*\* 「WIN」または「LOSE」画面を検知。

\- \*\*アクション:\*\* - 勝敗結果の記録。

&#x20; - デッキ推定エンジンのスコアに基づき「対面デッキ」を最終決定。

&#x20; - 戦績サマリーと詳細な棋譜ログをCSV等に出力。

\- \*\*遷移:\*\* State A へ戻る。



\## 5. 対面デッキ推定エンジン（スコアリングロジック）

\- 単純なカードの一致ではなく、環境デッキごとに定義された「キーカードの重み付けスコア」を用いて推論を行う。（例: 『ヘブンズ・ゲート』=10点、『ロージア』=5点など）

\- 試合終了時、最も合計スコアの高いデッキを対面デッキとして出力する。
