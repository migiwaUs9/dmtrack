# OCR改善 Phase 2: カード検知 + 画像ハッシュ

## 問題
- フレーム差分検知が誤爆 → カード非表示時にもOCRが走る（"回語語本生生語凍" が大量発生）
- OCRの精度が不十分（ゲームフォント対応の限界）

## 分析結果
- マスク画像の黒ピクセル率（black%）でカード有無を判別可能
  - カード表示時: black% > 5（テキストが存在）
  - 背景/盤面: black% = 0.0（テキストなし）

## 実装計画

### Step 1: カード表示バリデーション追加
- OCR実行前に、名前バナー領域のマスクを作成し black% をチェック
- 閾値未満（テキストなし）ならOCRをスキップ → 誤爆を根絶

### Step 2: pHash によるカード画像マッチング
- カード表示検知後、**カードイラスト領域**を切り出してパーセプチュアルハッシュ（pHash）を計算
- 既知のハッシュDBと照合 → 一致すればカード名確定
- DB にない場合:
  - OCRをフォールバックとして実行
  - カード画像を `cards_unknown/` に自動保存
  - ユーザーが後でラベル付け

### Step 3: インクリメンタルDB構築
- `card_hashes.json`: { hash文字列: カード名 } のマッピング
- 新カード検知時に自動で画像保存 + hash追加（未ラベル状態）
- GUIまたはCSVでラベル付けの仕組み

### 変更対象
- `tracker.py` — カード検知ロジック改修、pHash導入
- 新規: `card_hashes.json` — ハッシュDB
